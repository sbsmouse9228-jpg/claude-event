## [1] 문제 (Problem)

### 주요 문제 3가지

**① 주최자의 반복 수작업 과부하**
- 카카오톡 공지 발행 → 개별 DM으로 출석 확인 → 카풀 취합 → 정산 추적을 매 모임마다 수작업 반복
- 10명 모임 기준 한 주최자가 30~50건의 메시지 교환 발생

**② 참여자의 정보 파편화**
- 모임별 채팅방이 달라 공지를 놓치는 빈도 높음
- 정산 금액, 카풀 정보를 매번 직접 물어봐야 함

**③ 프로세스 추적 불가**
- 미응답자, 미납자를 엑셀/메모로 별도 관리
- 누가 송금했는지 확인하는 데 시간 소요

### 기존 대안
- 카카오톡 단체채팅 + 오픈채팅
- 구글 폼 (출석 확인 한정)
- 엑셀/메모 정산

## [2] 고객 세그먼트 (Customer Segments)

### 얼리 어답터 (Early Adopter)

**주요 타겟: 정기 소규모 모임 주최자**

| 구분 | 특징 |
|------|------|
| 연령 | 25~40세 직장인 |
| 모임 유형 | 정기 운동 모임 (수영, 헬스, 테니스), 친구 소셜 모임, 동호회 |
| 모임 규모 | **5~15명 (MVP 타겟)** |
| 빈도 | 주 1회 이상 정기 운영 |
| 특징 | 이미 카카오톡으로 관리 중 → 전환 비용 낮음 |

### 보조 타겟: 일반 참여자
- 여러 모임을 동시에 참여하는 직장인
- 간편한 링크 클릭으로 모든 처리를 원하는 사용자

## [3] 고유 가치 제안 (Unique Value Proposition)

### 핵심 메시지

> **"하나의 링크로 공지·출석·카풀·정산을 한 번에"**

### 주최자에게
- 공지 1회 발행 → 리마인드·정산 요청까지 자동 처리
- 실시간 출석 현황·정산 완료율 대시보드 제공

### 참여자에게
- 카카오 로그인 없이도 링크 하나로 이벤트 내용 확인
- 출석 확인, 카풀 신청, 정산 납부까지 한 화면에서 완결

## [4] 솔루션 (Solution)

| 문제 | 솔루션 |
|------|--------|
| 반복 공지 수작업 | 이벤트 생성 + 고유 초대 링크 자동 발급, 공지 발행 기능 |
| 출석 확인 개별 DM | 참가/불참/미정 선택 버튼, 미응답자 일괄 리마인드 |
| 카풀 조율 | 차량 등록 → 탑승 신청 → 수락/거절 → 자동 정보 전달 |
| 정산 추적 | 비용 입력 → 더치페이 자동 계산 → 납부 확인 → 미납자 알림 |
| 공지 누락 | 웹 푸시 알림 (D-1 자동 리마인드, 정산 등록 알림) |

## [5] 채널 (Channels)

### 획득 채널
- **카카오톡 링크 공유** — 초대 링크가 주요 바이럴 경로
- **비회원 접근 허용** — 링크 클릭만으로 이벤트 조회 가능 → 자연스러운 전환 유도

### 활성화 채널
- **웹 푸시 알림** — 이벤트 D-1, 정산 요청, 카풀 수락 알림 (MVP)
- **카카오 알림톡** — v2 도입 예정 (현재 비즈니스 채널 승인 검토 중)

### 유지 채널
- 이벤트 히스토리 (지난 모임 기록 조회)
- 주최자 대시보드 — 내 모임 목록으로 반복 접속 유도

## [6] 수익 구조 (Revenue Streams)

> MVP 단계에서는 수익화보다 사용자 확보에 집중

### 단기 (v1 이후)
- **프리미엄 플랜** (주최자 구독)
  - 무료: 월 이벤트 3회, 참여자 15명 이하
  - 유료: 무제한 이벤트, 반복 모임 자동 생성, 카카오 알림톡 발송

### 중장기 (v2+)
- **정산 수수료** — 결제 연동(토스페이/카카오페이) 시 거래 금액의 소수점 수수료
- **기업/동호회 플랜** — 팀 단위 대규모 모임 관리 기능 제공

## [7] 비용 구조 (Cost Structure)

### 고정 비용
| 항목 | 내용 |
|------|------|
| Vercel 호스팅 | Hobby → Pro 플랜 (트래픽 증가 시) |
| Supabase | Auth + PostgreSQL + Realtime (Free → Pro) |
| 도메인 / SSL | 연간 비용 |

### 변동 비용
| 항목 | 내용 |
|------|------|
| 웹 푸시 알림 | Web Push API (MVP 단계 무료) |
| 카카오 알림톡 | 건당 발송 비용 (v2 도입 시) |
| 카카오 OAuth | 현재 무료, 트래픽 제한 확인 필요 |

### 개발 비용
- 초기 개발: 6주 로드맵 (1인 개발 기준)
- 운영 유지: 월 버그 수정 및 기능 개선

## [8] 핵심 지표 (Key Metrics)

### MVP 성공 기준

| 지표 | 목표 | 측정 방법 |
|------|------|---------|
| 이벤트 생성 수 | 베타 기간 내 **30개 이상** | DB 집계 |
| 초대 링크 클릭률 | 발송 대비 **70% 이상** | 링크 접근 로그 |
| 출석 확인 완료율 | 이벤트당 참여자 **80% 이상** | attendance 테이블 |
| 정산 납부 완료율 | 정산 등록 이벤트의 **90% 이상** | settlement_payments 집계 |
| 주최자 재사용률 | **50%** 이상이 2회 이상 이벤트 생성 | 사용자별 이벤트 count |

### 선행 지표 (Leading Indicator)
- 이벤트 생성 후 초대 링크 공유까지 소요 시간 (낮을수록 UX 우수)
- 참여자 1인당 출석 확인까지 클릭 수 (3회 이내 목표)

## [9] 경쟁 우위 (Unfair Advantage)

### 쉽게 복제하기 어려운 강점

**① 카카오 생태계 네이티브 통합**
- 한국 사용자 97%가 사용하는 카카오 소셜 로그인
- 초대 링크가 카카오톡에서 자연스럽게 퍼지는 바이럴 구조

**② 마찰 없는 비회원 진입**
- 링크 클릭 → 로그인 없이 이벤트 조회 가능
- 출석 확인 필요 시 카카오 로그인 후 자동 리다이렉트 → 가입 전환율 극대화

**③ 소규모 모임 특화 경량 UX**
- 대형 이벤트 플랫폼(이벤터스 등)과 달리 5~15명 일상 모임에 최적화
- 모바일 퍼스트 설계로 채팅방에서 바로 전환 가능

**④ 올인원 워크플로우**
- 공지 → 출석 → 카풀 → 정산을 하나의 화면에서 완결
- 경쟁 서비스는 각 기능을 별도 앱으로 파편화

## 가정 검증 우선순위

| 순위 | 핵심 가정 | 검증 방법 |
|------|---------|---------|
| 1 | 주최자가 카카오톡 대신 별도 도구를 사용할 의향이 있다 | 베타 유저 5명 인터뷰 |
| 2 | 비회원 링크 접근이 가입 전환으로 이어진다 | 링크 → 로그인 퍼널 분석 |
| 3 | 정산 기능이 재사용률을 높이는 핵심 후크다 | 정산 사용 유저 vs 미사용 유저 재방문율 비교 |
| 4 | 카풀 매칭이 실제로 활용된다 | 카풀 등록 이벤트 비율 측정 |
