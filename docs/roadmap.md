## 버전 개요

| 버전 | 목표 | 기간 |
|------|------|------|
| **v0.1** | 기반 세팅 + 이벤트 생성 + 초대 링크 | 1~2주차 |
| **v0.2** | 공지 발행 + 출석 확인 + 대시보드 | 2~3주차 |

## v0.1 — 기반 세팅 & 이벤트 생성 (1~2주차)

**목표:** 주최자가 이벤트를 만들고 초대 링크를 카카오톡에 공유할 수 있다

### 인프라 & 프로젝트 세팅

- [ ] Next.js 14 (App Router) + TypeScript 프로젝트 초기화
- [ ] Tailwind CSS 설정
- [ ] Supabase 프로젝트 생성 및 환경변수 연결
- [ ] Vercel 배포 파이프라인 연결 (main 브랜치 자동 배포)
- [ ] ESLint / Prettier 설정

### 인증 (FR-AUTH)

- [ ] Supabase Auth — 카카오 OAuth 2.0 연동
- [ ] 로그인 / 로그아웃 플로우 구현
- [ ] 로그인 상태에 따른 라우트 보호 미들웨어
- [ ] 초대 링크 진입 시 로그인 후 원래 페이지로 리다이렉트

### DB 스키마

- [ ] `users` 테이블 마이그레이션
- [ ] `events` 테이블 마이그레이션 (invite_token 포함)
- [ ] Row Level Security (RLS) 정책 설정

### 이벤트 생성 (FR-EVENT)

- [ ] 이벤트 생성 폼 (`/event/create`)
  - 제목, 날짜/시간, 장소 (필수)
  - 최대 인원, 설명, 참가비 (선택)
- [ ] 이벤트 생성 시 UUID 기반 초대 토큰 자동 발급
- [ ] 초대 링크 복사 버튼 (클립보드)
- [ ] 초대 링크 진입점 구현 (`/invite/[token]`)
  - 비회원도 이벤트 정보 조회 가능
- [ ] 이벤트 수정 / 삭제

### 랜딩 & 대시보드 뼈대

- [ ] 랜딩 페이지 (`/`) — 서비스 소개 + 카카오 로그인 버튼
- [ ] 내 모임 목록 페이지 (`/dashboard`) — 주최 / 참여 탭 구조만

**완료 기준:** 주최자가 이벤트를 만들고 초대 링크를 공유하면 비회원이 이벤트 내용을 확인할 수 있다

---

## v0.2 — 공지 발행 & 출석 확인 (2~3주차)

**목표:** 주최자가 공지를 보내고 참여자 출석 현황을 실시간으로 확인할 수 있다

### DB 스키마 추가

- [ ] `event_participants` 테이블 마이그레이션
- [ ] `notices` 테이블 마이그레이션

### 출석 확인 (FR-ATTEND)

- [ ] 참여자 출석 상태 선택 UI (참가 / 불참 / 미정)
- [ ] 최대 인원 초과 시 참가 신청 불가 처리
- [ ] 주최자 — 실시간 출석 현황 표시 (N/전체명 참가)
- [ ] 미응답자 목록 표시
- [ ] 미응답자 리마인드 알림 일괄 발송 버튼

### 공지 (FR-NOTICE)

- [ ] 주최자 공지 작성 폼
- [ ] 공지 목록 표시 (최신순)
- [ ] 공지 발행 시 참여자 웹 푸시 알림 발송

### 주최자 관리 패널

- [ ] `/event/[id]/manage` 페이지
  - 참여자 현황 테이블
  - 공지 작성
  - 이벤트 정보 수정

### 이벤트 메인 페이지 완성

- [ ] `/event/[id]` 참여자 뷰
  - 이벤트 정보
  - 공지 목록
  - 출석 확인 버튼
  - 카풀 / 정산 현황 요약 (v0.3, v0.4에서 채워질 슬롯)

### 대시보드 완성

- [ ] 내 모임 목록 — 주최 / 참여 탭
- [ ] 이벤트 상태 배지 표시 (예정 / 진행중 / 완료 / 취소)

**완료 기준:** 주최자가 공지를 발행하고, 출석 확인 현황을 실시간으로 볼 수 있으며 미응답자에게 리마인드를 보낼 수 있다

## v0.3 — 카풀 매칭 (3~4주차)

**목표:** 참여자끼리 차량을 등록하고 탑승을 신청해 카풀 그룹이 자동 구성된다

### DB 스키마 추가

- [ ] `carpool_cars` 테이블 마이그레이션
- [ ] `carpool_requests` 테이블 마이그레이션

### 카풀 기능 (FR-CARPOOL)

- [ ] 차량 등록 폼
  - 출발 위치, 출발 시간, 탑승 가능 인원
- [ ] 등록된 차량 목록 카드 표시
- [ ] 탑승 신청 버튼 (잔여석 표시)
- [ ] 차량 제공자 — 신청자 수락 / 거절 UI
- [ ] 수락 시 탑승자에게 알림 (차량 제공자 연락처, 출발 장소 포함)
- [ ] 카풀 현황 페이지 (`/event/[id]/carpool`)
  - 전체 카풀 그룹 구성 한눈에 보기
  - 미배정 참여자 목록

### 이벤트 메인 업데이트

- [ ] 카풀 현황 요약 위젯 (배정 완료 N명 / 미배정 M명)

**완료 기준:** 차량 등록 → 탑승 신청 → 수락 → 카풀 그룹 자동 구성 및 알림 전송까지 동작한다

---

## v0.4 — 정산 시스템 (4~5주차)

**목표:** 비용 항목을 입력하면 참여자별 납부 금액이 자동 계산되고, 미납자에게 알림을 보낼 수 있다

### DB 스키마 추가

- [ ] `settlements` 테이블 마이그레이션
- [ ] `settlement_payments` 테이블 마이그레이션

### 정산 기능 (FR-SETTLE)

- [ ] 비용 항목 입력 폼
  - 항목명, 금액, 부담 대상 (전원 / 선택 멤버)
- [ ] 더치페이 자동 계산 로직
  - 원 단위 반올림 처리 (마지막 인원이 나머지 부담)
- [ ] 참여자별 납부 금액 표시
- [ ] 주최자 계좌 정보 등록 및 표시
- [ ] 참여자 '납부 완료' 체크 버튼
- [ ] 정산 완료율 표시 (N/M명 완료)
- [ ] 미납 참여자에게 알림 발송 버튼
- [ ] 정산 상세 페이지 (`/event/[id]/settle`)

### 알림 완성 (FR-NOTIFY)

- [ ] D-1 자동 리마인드 — Supabase Edge Function + Cron
- [ ] 정산 등록 시 개인별 납부 금액 알림

### 이벤트 메인 업데이트

- [ ] 정산 현황 요약 위젯 (N/M명 납부 완료)

**완료 기준:** 주최자가 비용을 입력하면 개인별 금액이 계산되고 미납자 알림 발송까지 가능하다

## v1.0 — MVP 완성 & 배포 (5~6주차)

**목표:** 실제 모임에서 사용 가능한 수준으로 안정화하고 공개 배포한다

### QA & 버그 수정

- [ ] 전체 유저 플로우 E2E 테스트
  - 주최자 플로우: 이벤트 생성 → 공지 → 출석 확인 → 카풀 → 정산
  - 참여자 플로우: 초대 링크 → 로그인 → 출석 확인 → 카풀 신청 → 납부 확인
- [ ] 모바일 반응형 UI 점검 (375px, 390px, 414px)
- [ ] RLS 보안 정책 최종 검토
- [ ] 엣지 케이스 처리 (인원 초과, 카풀 중복 신청, 정산 재계산 등)

### 성능 & UX 개선

- [ ] 페이지 로드 LCP 3초 이내 확인
- [ ] 로딩 스켈레톤 UI 적용
- [ ] 에러 상태 / 빈 상태 UI 처리

### 배포

- [ ] Vercel 프로덕션 환경 변수 설정
- [ ] 커스텀 도메인 연결 (선택)
- [ ] Supabase 프로덕션 프로젝트 분리

### 베타 테스트

- [ ] 실제 모임 1~2개에서 테스트 운영
- [ ] 피드백 수집 및 크리티컬 버그 수정

**완료 기준:** 실제 모임에서 처음부터 끝까지 MOA만으로 운영 완료

## v2.0 — 성장 기능 (MVP 이후)

> MVP 지표 달성 후 우선순위 결정

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 반복 이벤트 | 매주/매월 자동 이벤트 생성 | 높음 |

| 결제 연동 | 토스페이 / 카카오페이 직접 송금 | 중간 |
| 모임 공개 모집 | 새 멤버 모집 및 검색 | 낮음 |
| 모바일 앱 | React Native 또는 PWA | 낮음 |
